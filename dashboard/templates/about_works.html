HOW THE TOOL WORKS

 

STRUCTURE

 

       For every 2-month cycle, the national dataset of facility ARV orders is fed into the system by the administrator

o   This covers data for a comprehensive list of 1800+ ART facilities (which is based on the DHIS2 master list)

o   It is important to note, however, that, through poor practice, not all of these 1800+ facilities will typically submit an order in any given cycle

-       The facility-level data is interrogated via c.20 tests that have been constructed within the system to assess quality. These tests are outlined in detail on the Test Explanation page

-       Any given test is run on data from a single cycle or from two consecutive ones

-       Certain tests, due to their nature, cannot assess the whole order form and are thus run on data from three Sample Formulations. These three formulations have been chosen for their importance in Adult and Paediatric treatment:

o   Sample Formulation 1: Adult TDF/3TC/EFV 300/300/600mg - the principal Adult 1st Line 3-dose combination therapy

o   Sample Formulation 2:Paediatric ABC/3TC 60/30mg - the principal Paediatric 1st Line NRTI combination drug

o   Sample Formulation 3:Paediatric EFV 200mg - the principal Paediatric 1st Line NNRTI drug

 

SCORING

 

-       All 1800+ facilities are run through the c.20 tests and score either a "Pass", "Fail" or "Insufficient Data" for each test (though these outcomes do vary slightly for some tests)

o   "Pass" denotes a facility's success in the particular test and is a signal of good order quality

o   "Fail" denotes a facility's failure in the particular test and is a signal of poor order quality

o   "Insufficient data" denotes the lack of enough facility data for the particular test to be run. This results, typically, from a facility not submitting an order form in the cycle in question and thus also reflects failure to an extent

 

SYSTEM DISPLAY

 

-       The system presents these "Pass", "Fail" and "Insufficient Data" scores in aggregated and granular ways through various charts and tables. In doing so it enables the user to identify important themes in ARV ordering performance

-       This data is presented to the user predominantly via three interfaces:

1.     Dashboard - displays test performance predominantly in a number of charts, i.e. at an aggregated level (across all facilities in the user's domain) and over time so a trend can be observed. Also displays ranking lists that enable the user to see which IPs/Districts/Warehouse/Facilities are best and worst performing within their domain

2.     Facility Report - displays the granular performance data for each facility in the user's domain across all tests. Each row of the table represents an individual facility and comprises facility name, district, IP, warehouse and all test outcomes. The table enables filtering across all key parameters, switching between different cycles of data and searching for particular facilities

3.     Raw Data Pop-up - Clicking on any "Pass" or "Fail" in the Facility Report will display a pop-up visualisation of the raw order data that is contributing to the test outcome (i.e. the correct data behind a "Pass" or the offending data behind a "Fail")
